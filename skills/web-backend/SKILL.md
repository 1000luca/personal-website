# Web Backend

## Overview
This skill defines data models and APIs, and implements integrations required for website functionality.

## When To Use
- Forms, auth, dashboards, or dynamic data are required
- CMS, payments, email, or third-party integrations are needed
- A site needs server-side logic or persistence

## Workflow
1. Define data model and ownership.
2. Specify API endpoints and contracts.
3. Choose storage and migrations strategy.
4. Implement auth and authorization if needed.
5. Integrate third-party services.
6. Add validation, rate limits, and logging.
7. Document usage and error handling.

## Inputs Needed
- Data entities and fields
- Auth requirements and roles
- Integration list and credentials
- Non-functional requirements, SLA, and scale

## Output Expectations
- Clear API spec with examples
- Secure defaults and validation
- Observable errors and logs
