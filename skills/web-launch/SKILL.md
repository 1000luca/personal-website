# Web Launch

## Overview
This skill validates quality and readiness for production, then deploys with monitoring.

## When To Use
- Pre-launch or before a major update
- When performance or SEO must be validated
- When adding analytics or observability

## Workflow
1. QA: functional checks, cross-browser, mobile.
2. Accessibility: keyboard, focus, contrast, ARIA usage.
3. Performance: Lighthouse, asset sizes, caching.
4. SEO: metadata, sitemap, robots, structured data.
5. Analytics: events, goals, consent, privacy.
6. Deployment: build, environment, CI/CD, rollback.
7. Post-launch: monitoring and error tracking.

## Launch Checklist
- No broken links or missing assets
- Core pages load within performance budget
- Sitemap and robots are correct
- Tracking is verified
- Rollback plan is documented
